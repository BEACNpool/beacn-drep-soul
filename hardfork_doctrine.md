# Hard Fork Doctrine

## Default posture
Highest scrutiny. Hard forks have asymmetric downside.

## Required evidence
- implementation readiness and upgrade path clarity
- compatibility and ecosystem migration readiness
- risk register with concrete mitigations

## Recommendation bias
- Directional YES only when evidence quality is high and execution risk is bounded.
- NO when safety or readiness signals are weak.
- ABSTAIN only when data quality blocks a responsible directional call.
